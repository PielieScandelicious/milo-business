\subsection{Mission Statement}

\begin{highlightbox}
\textit{``Empower health-conscious consumers to understand not just \textbf{how much} they spend, but \textbf{whether they're spending on healthy choices} --- while building an ethically-sourced, privacy-compliant consumer data platform.''}
\end{highlightbox}

\subsection{Three-Sentence Summary}

\textbf{MILO} is a production-ready iOS application powered by a FastAPI backend that combines AI-driven receipt scanning (Veryfi OCR), proprietary health scoring (0-5 scale), and an intelligent chat assistant (``Milo'' --- powered by Gemini 2.0 Flash) into a unified wellness-aware spending platform.

The technical stack is complete: native SwiftUI frontend with Firebase authentication, actor-based service architecture, SSE streaming chat, and a modular Python backend with 17-category classification and PostgreSQL persistence.

With \$400K-\$600K in seed funding, MILO targets 25,000 monthly active users by Year 3 and a minimum \$15M valuation at exit.

\section{Technical Architecture}
%==============================================================================

\subsection{System Architecture Overview}

\begin{highlightbox}[title=\textbf{MILO Technical Stack v4.0}]
\textbf{iOS Frontend:} SwiftUI (iOS 17+), Firebase Auth, StoreKit 2, Actor-based Services

\textbf{Backend API:} FastAPI 0.109.0 (Python 3.11), PostgreSQL + SQLAlchemy 2.0, Async/Await

\textbf{AI/ML Services:} Veryfi OCR v8, Gemini 2.0 Flash, 17-Category Classification

\textbf{Infrastructure:} Railway Hosting, Firebase Storage, End-to-End Encryption
\end{highlightbox}

\subsection{Backend Architecture (FastAPI)}

The backend follows a clean layered architecture:

\begin{itemize}
    \item \textbf{API Layer} (\texttt{app/api/v2/}): Route handlers, request validation
    \item \textbf{Service Layer} (\texttt{app/services/}): Business logic, external API integration
    \item \textbf{Repository Layer} (\texttt{app/db/repositories/}): Data access, SQL queries
    \item \textbf{Model Layer} (\texttt{app/models/}): SQLAlchemy ORM definitions
\end{itemize}

Key services:
\begin{itemize}
    \item \texttt{receipt\_processor\_v2.py} --- OCR $\rightarrow$ Categorize $\rightarrow$ Save pipeline
    \item \texttt{veryfi\_service.py} --- Veryfi OCR integration
    \item \texttt{categorization\_service\_gemini.py} --- AI categorization \& health scores
    \item \texttt{dobby\_chat\_service\_gemini.py} --- Streaming chat assistant
    \item \texttt{rate\_limit\_service.py} --- Usage tracking (15 receipts, 100 messages/mo)
\end{itemize}

\subsection{iOS Frontend Architecture (SwiftUI)}

The iOS app uses a modern MVVM architecture:

\begin{itemize}
    \item \textbf{Views} (\texttt{Views/}): SwiftUI views with dark-mode design
    \item \textbf{ViewModels} (\texttt{ViewModels/}): ObservableObject state management
    \item \textbf{Services} (\texttt{Services/}): Actor-based API clients
    \item \textbf{Models} (\texttt{Models/}): Codable data structures
\end{itemize}

Key components:
\begin{itemize}
    \item \texttt{AuthenticationManager} --- Firebase Auth, token sharing via Keychain
    \item \texttt{AnalyticsAPIService} --- Actor-based async/await API client
    \item \texttt{MiloAIChatService} --- SSE streaming via AsyncThrowingStream
    \item \texttt{ReceiptUploadService} --- Multipart form data upload
\end{itemize}
