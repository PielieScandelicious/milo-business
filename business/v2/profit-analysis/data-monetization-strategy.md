## Consent Architecture

### Tiered Consent Model

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                        CONSENT TIER STRUCTURE                               │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  TIER 1: Basic (Required)                                                   │
│  ━━━━━━━━━━━━━━━━━━━━━━━                                                   │
│  • App functionality data                                                   │
│  • Anonymous usage analytics                                               │
│  • No external sharing                                                     │
│                                                                             │
│  TIER 2: Enhanced (Optional - Target: 60% opt-in)                          │
│  ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━                             │
│  • Aggregated trend reports                                                │
│  • Category-level insights                                                 │
│  • No individual identification                                            │
│                                                                             │
│  TIER 3: Full Panel (Optional - Target: 40% of Tier 2)                     │
│  ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━                        │
│  • Item-level data sharing                                                 │
│  • Panel participation                                                     │
│  • Incentivized ($5/month credit or equivalent)                           │
│                                                                             │
│  CONSERVATIVE TARGET: 40% of users at Tier 3                               │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

| Right to withdraw | ✅ Designed | One-click revocation |

### Data Processing Safeguards

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                         DATA PROTECTION MEASURES                            │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  ✓ Anonymization: No PII in B2B data products                              │
│  ✓ Aggregation: Minimum cohort sizes (n≥100)                               │
│  ✓ Encryption: Data encrypted at rest and in transit                       │
│  ✓ Access Control: Role-based access for B2B partners                      │
│  ✓ Audit Trail: Complete logging of data access                            │
│  ✓ DPAs: Data Processing Agreements with all buyers                        │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```
