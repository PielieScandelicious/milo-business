## Key Takeaways

### 1. Different Value Propositions

| Competitor | Core Value | User Motivation |
|------------|------------|-----------------|
| **Fetch** | Easy points | "Free money for shopping" |
| **Ibotta** | Real cashback | "Save money on groceries" |
| **Receipt Hog** | Fun & games | "Make scanning entertaining" |
| **MILO** | Health insights | "Understand my spending & health" |

### 2. MILO's Differentiation Strategy

MILO should **not compete on rewards** but instead:

```diff
+ Position as "insights" app, not "rewards" app
+ Emphasize health scoring (unique feature)
+ Target different user psychographic (health-conscious)
+ Highlight AI assistant value
+ Market privacy-first approach
```

### 3. Coexistence Opportunity

Many users scan receipts into **multiple apps**:

```
User Behavior Pattern:
├── Scan into Fetch (for points)
├── Scan into Ibotta (for cashback)
└── Scan into MILO (for insights)

Opportunity: Users can use MILO alongside reward apps
```

### 4. Threat Summary

| Competitor | Threat Level | Rationale |
|------------|--------------|-----------|
| **Fetch** | 🟡 Medium | Could add insights; massive scale |
| **Ibotta** | 🟡 Medium | B2B data ambitions; public resources |
| **Receipt Hog** | 🟢 Low | Different focus, smaller scale |
| **Expensify** | 🟢 Low | Business-focused |

---

## Recommendations

1. **Avoid reward competition** - Do not add points/cashback (loses differentiation)
2. **Embrace complementary positioning** - "Use with Fetch/Ibotta for rewards + insights"
3. **Lead with health angle** - Primary differentiator no competitor has
4. **Move fast on AI features** - Prevent Fetch from catching up
5. **Target Yuka users** - Health-conscious users who already scan products
